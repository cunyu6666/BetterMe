# PRD：BetterMe 长期维护与演进

## 1. 简介

BetterMe 是一个个人健康管理 Web 应用（React 19 + TypeScript + Vite），包含习惯追踪、目标管理、健康记录、学习进度、医疗自护计划等模块。当前处于早期原型阶段——无 Git、无测试、无 CI/CD、数据仅存 localStorage。

本文档定义了从原型走向可持续维护的完整路线图，覆盖工程基建、数据层升级、功能演进和用户体验四个维度。

## 2. 目标

- **G-1**：建立完整的工程基建（Git、测试、CI/CD、代码规范），使项目可持续迭代
- **G-2**：将数据持久化从 localStorage 迁移到 insforge 后端，支持数据备份与恢复
- **G-3**：增强现有功能模块，提升日常使用效率
- **G-4**：优化用户体验，包括 PWA 支持、离线能力、移动端适配

## 4. 功能需求

- **FR-1**：Git 版本控制——项目必须使用 Git 管理源码，含合理的 `.gitignore`
- **FR-2**：自动化测试——至少覆盖核心工具函数（`storage.ts`、`timer.ts`），目标覆盖率 ≥ 60%
- **FR-3**：CI/CD 流水线——每次提交自动执行 lint + 类型检查 + 测试，main 分支自动部署到 Netlify
- **FR-4**：insforge 后端——所有数据通过 insforge SDK 持久化，保留 localStorage 作为离线缓存
- **FR-5**：数据迁移——提供从 localStorage 到 insforge 的一次性迁移路径
- **FR-6**：数据导出——支持将全部数据导出为 JSON
- **FR-7**：习惯自定义频率——支持每日/每周N次/自定义间隔
- **FR-8**：目标里程碑——支持带日期的关键节点追踪
- **FR-9**：健康趋势可视化——睡眠、运动、心情的周期性趋势图表
- **FR-10**：PWA——Manifest + Service Worker，支持离线使用和安装到主屏幕
- **FR-11**：深色模式——CSS 变量驱动的主题系统
- **FR-12**：代码质量——迁移到 Tailwind CSS 4，消除 inline style，lint 零警告

## 5. 非目标（范围外）

以下内容**明确不在**本 PRD 范围内：

- **不做多用户系统**——没有注册/登录/权限，始终是个人工具
- **不做社交功能**——没有分享、排行榜、社区
- **不做移动端原生 App**——不开发 iOS/Android 原生版本，仅 PWA
- **不做 AI 功能**——不引入 AI 推荐、智能分析等
- **不做计划模块的通用化**——鼻炎管理和手腕复健计划保持硬编码现状
- **不做支付/商业化**——无订阅、无付费功能
- **不做国际化**——仅支持中文界面

## 6. 设计考量

- 采用 Tailwind CSS 4 作为样式方案，通过 `@tailwindcss/vite` 插件集成
- 深色模式通过 Tailwind 的 `dark:` 前缀 + CSS 变量实现
- 新增图表使用轻量级方案（如 recharts 或纯 SVG），避免引入 ECharts 等重型库
- PWA 图标复用现有 `favicon.svg`，生成多尺寸版本
- 移动端底部 Tab 导航保持现有交互模式

## 7. 技术考量

- **insforge 集成**：
  - 认证：`npx @insforge/cli login --user-api-key <key>`
  - 项目关联：`npx @insforge/cli link --project-id 65f3c914-d253-4f34-adad-5f22314a4a8e`
  - 使用 InsForge CLI 和 skills 管理后端任务（数据模型、API 等）
- **离线策略**：Service Worker 采用 Cache-First 策略缓存静态资源，API 请求在离线时降级到 localStorage 缓存
- **CSS 迁移**：从 `index.css`（~550 行）迁移到 Tailwind CSS 4，通过 `@tailwindcss/vite` 插件集成，保留 CSS 变量用于主题色
- **测试选型**：Vitest 原生支持 Vite，零配置集成；@testing-library/react 测试组件行为
- **部署**：Netlify，连接 GitHub 仓库，`npm run build` → `dist/`，push 到 main 自动部署
